2. ğŸ§ª Creating Local Kubernetes Clusters

Today I learned two tools for creating local Kubernetes clusters:


=========================================
   ğŸ³ KIND (Kubernetes IN Docker)
=========================================

- Creates Kubernetes clusters using Docker containers
- Lightweight and fast
- Good for testing, CI/CD pipelines, GitHub Actions
- Cluster runs entirely inside Docker

Basic command:

kind create cluster --name dev-cluster

This creates:
- Control plane as a Docker container
- Worker nodes as Docker containers


=========================================
   ğŸ¡ Minikube
=========================================

- A single-node Kubernetes cluster for local development
- Uses a VM or Docker driver
- Great for learning Kubernetes resources (pods, services, deployments)

Start a cluster:

minikube start

Check status:

minikube status

Minikube also provides:
- Built-in dashboard
- Add-ons (ingress, metrics-server, registry)
