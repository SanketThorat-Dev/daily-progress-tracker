2. ğŸ Docker Scout CVEs
=========================================

A **CVE (Common Vulnerabilities and Exposures)** is a known security flaw
in a software component.

Docker Scout scans image layers and identifies:
- Vulnerable OS packages
- Vulnerable libraries
- Insecure base images
- Outdated dependencies

CVEs come with severity levels:
- Critical
- High
- Medium
- Low

-----------------------------------------

-----------------------------------------

Scout also shows:
- The exact layer where the CVE exists
- How to fix it (e.g., use a newer base image)

=========================================

`docker init` is a Docker command that automatically generates Docker
configuration files for a project.

It can create:
- Dockerfile
- .dockerignore
- compose.yaml (optional)

-----------------------------------------
ğŸ§© Why use docker init?
-----------------------------------------
- Quickly containerizes any project
- Detects language automatically (Node, Python, Go, Java, etc.)
- Creates production-ready Dockerfiles
- Saves time and reduces manual setup

-----------------------------------------
ğŸ“˜ Example Workflow
-----------------------------------------
Run:
    docker init

Docker will scan the project and generate files like:

Dockerfile:
-----------------------------------------
FROM node:18-alpine
WORKDIR /app
COPY package*.json .
RUN npm install
COPY . .
CMD ["npm", "start"]
-----------------------------------------

This gives you a clean and optimized starting point for containerization.
