1. ğŸ—‚ï¸ Kubernetes Namespaces
=========================================

Namespaces are used to logically divide a Kubernetes cluster.

They help in:
- Organizing resources
- Isolating environments
- Applying access control
- Managing multi-team and multi-tenant setups

Each Kubernetes cluster comes with built-in namespaces:
- default
- kube-system
- kube-public
- kube-node-lease

Why namespaces matter:
- Prevent resource name conflicts
- Allow different teams/projects to work in the same cluster
- Enable resource quotas
- Provide security boundaries

A **Pod** is the smallest deployable object in Kubernetes.
It represents:
- A single container
OR
- Multiple tightly-coupled containers (sidecar pattern)

Pods share:
- Network (same IP)
- Storage (volumes)
- Linux namespaces

Pods are short-lived and not self-healing, so usually we donâ€™t create
